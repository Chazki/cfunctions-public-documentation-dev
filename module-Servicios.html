    Methods

    uploadClientOrders(enterpriseKey, orders) → {Response}

    Este endpoint permite registrar un delivery. Dado que en el los datos se puegen agrupar en un arreglo, se pueden registrar varios deliveries

    Los datos deben de ser enviados en el protocolo POST y en formato del body request en JSON.

    URL: https://us-central1-chazki-link-dev.cloudfunctions.net/uploadClientOrders

    Nota: La estructura de los objetos del array "orders" es igual a la proporcionada en el esquema para la carga CSV. Puede que el siguiente listado de parametros tenga mas o menos campos. En todo caso, utilize los campos del esquema CSV.

    Parameters:
    Name Type Description
    enterpriseKey String

    Codigo unico de autorizacion que identifica al cliente.
    orders Array.<Object>

    Array de objetos con la informacion de las ordenes que se quieren crear para el cliente.

    Properties
    Name Type Attributes Description
    trackCode String

    Codigo de la orden.
    paymentMethodID String

    Metodo de pago de la orden, puede tomar los valores de PAGADO, COD ,CCOD.
    paymentProofID String

    Documento de pago, puede tomar los valores FACTURA, BOLETA y GUIA.
    serviceID String

    Codigo del servicio. Puede tomar los valores de "REGULAR", "EXPRESS" O "PROGRAMADO", dependiendo del tipo de servicio deseado.
    packageEnvelope String

    Tipo de envoltura del paquete.
    packageWeight Int

    Peso del paquete.
    packageSizeID String

    Unidad de medida del pedido (XS, S, M, L)
    packageQuantity String

    Cantidad de paquetes de la orden.
    productDescription String

    Descripcion de los productos.
    productPrice Float <optional>

    Precio aproximado del producto.
    reverseLogistic String <optional>

    Puede tomar los valores de "SI" o "NO".
    crossdocking String <optional>

    Campo que indica si la orden es crossdocking. Puede tomar los valores de "SI" o "NO".
    pickUpBranchID String <optional>

    Codigo del almacen a recolectar los paquetes.
    pickUpAddress String

    Direccion del cliente.
    pickUpPostalCode String <optional>

    Codigo postal de la direccion a recolectar.
    pickUpAddressReference String

    Referencias de la direccion del cliente.
    pickUpPrimaryReference String

    Distrito de la direccion del cliente.
    pickUpSecondaryReference String

    Provincia de la direccion del cliente.
    pickUpNotes String

    Notas para la recoleccion del paquete.
    pickUpContactName String <optional>

    Nombres de la persona de contacto para la recoleccion.
    pickUpContactPhone String <optional>

    Numero telefonico de la persona de contacto de la recoleccion.
    pickUpContactDocumentTypeID String <optional>

    Tipo de documento de la persona de contacto para la recoleccion. Puede tomar valores de "DNI", "PAS", "PTP", "CR" y "CE".
    pickUpContactDocumentNumber String <optional>

    Numero del documento de identidad de la persona de contacto para la recoleccion.
    pickUpContactEmail String <optional>

    Email para notificaciones de recoleccion.
    dropBranchID String <optional>

    Codigo del almacen a entregar.
    dropAddress String

    Direccion de entrega del cliente final.
    dropPostalCode String <optional>

    Codigo postal de la direccion a recolectar.
    dropAddressReference String

    Referencias de la direccion del cliente final.
    dropPrimaryReference String

    Distrito de la direccion del cliente final.
    dropSecondaryReference String

    Provincia de la direccion del cliente final.
    dropNotes String

    Notas para la entrega del paquete.
    dropContactName String <optional>

    Nombres de la persona de contacto para la entrega.
    dropContactPhone String <optional>

    Numero telefonico de la persona de contacto de la entrega.
    dropContactDocumentTypeID String <optional>

    Tipo de documento de la persona de contacto para la entrega. Puede tomar valores de "DNI", "PAS", "PTP", "CR" y "CE".
    dropContactDocumentNumber String <optional>

    Numero del documento de identidad de la persona de contacto para la entrega.
    dropContactEmail String <optional>

    Email para notificaciones de la entrega.
    pickUpPoint Array.<Float> <optional>

    Array de coordenadas de la direccion de recoleccion. Debe tener dos elementos que indican la latitud y longitud
    dropPoint Array.<Float> <optional>

    Array de coordenadas de la direccion de entrega. Debe tener dos elementos que indican la latitud y longitud
    Throws:

    Si no se proporciona el enterpriseKey o el nombre correcto de las etiquetas, los pedidos no seran creados.

    Returns:
    Response -

    Objeto de respuesta con el estado de la creacion de las ordenes.

     {
   "enterpriseKey": "XXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX",
   "orders": [
     {
       "trackCode": "ABCDEFI",

    URL: https://us-central1-chazki-link-dev.cloudfunctions.net/fnTrackOrderChazkiPosition

    Parameters:
    Name Type Description
    code String

    Código track del delivery
    enterpriseKey String

    Codigo unico de autorizacion que identifica al cliente.
    Examples

    Envio de datos en formato cURL

      curl --request GET 'https://us-central1-chazki-link-dev.cloudfunctions.net/fnTrackOrderChazkiPosition?code=XXXXXXXXX&enterpriseKey=YYYYYYYYYYYYYYY'

    createPDFByOrders(enterpriseKey, filters) → {Response}

    Endpoint que permite generar las etiquetas en el formato estandar de Chazki

    El archivo a generar esta en extencion PDF.

    Parameters:
    Name Type Description
    enterpriseKey String

    Codigo unico de autorizacion que identifica al cliente.
    filters Object

    Objeto con los filtros para generar las etiqueas las etiquetas en formato PDF.

    Properties
    Name Type Description
    trackCodes Array.<String>

    Trackcode o codigo de seguimiento de la orden
    Throws:

    Si no se proporciona el enterpriseKey o el nombre correcto de las etiquetas, no se generara un PDF.

    Returns:
    Response -

    Objeto de respuesta con la direccion URL del PDF.

    Examples

    Parametros enviados en el body del request en formato JSON

     {
   "enterpriseKey": "XXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX",
   "filters": {
     "trackCodes":["XXXXXXXX","YYYYYYYYYY"]
   }
 }

    Respuesta

     {
   "success": true,
   "file": "https://storage.googleapis.com/download/storage/v1/b/xxxxxxxxxxxxx/yyyyyyy"
 }
