    uploadClientOrders(enterpriseKey, orders) → {Response}

    This endpoint allows to register a delivery. Since the data can be grouped into an array, multiple deliveries can be registered

    Data must be sent in POST protocol and in the body request format in JSON.

    URL: https://us-central1-chazki-link-dev.cloudfunctions.net/uploadClientOrders

    Note: The objects structure in the "orders" array is the same as the one provided in the schema for CSV loading. The following list of parameters may have more or fewer fields. In any case, use the fields from the CSV scheme.

    Parameters:
    Name Type Description
    enterpriseKey String

    Unique authorization code that identifies the customer.
    orders Array.<Object>

    Objects array containing the orders information required for the client.

    Properties
    Name Type Attributes Description
    trackCode String

    Order code.
    paymentMethodID String

    Order payment method, It can take the next values COD, PAGADO, CCOD, PAY.
    paymentProofID String

    Payment document, It can take the next values BOLETA, GUIA and FACTURA.
    serviceID String

    Service code. It can take the next values "REGULAR", "EXPRESS" or "PROGRAMMED", depending on the desired service type.
    packageEnvelope String

    Package wrap type.
    packageWeight Int

    Package weight.
    packageSizeID String

    Order measure unit (XS, S, M, L).
    packageQuantity String

    Order packages Quantity.
    productDescription String

    Products description.
    productPrice Float <optional>

    Approximate product price.
    reverseLogistic String <optional>

    It can take the next values 'YES' or 'NO'
    crossdocking String <optional>

    Field indicating if the order is crossdocking. It can take the next values "YES" or "NO".
    pickUpBranchID String <optional>

    Warehouse code where the packages are collected.
    pickUpAddress String

    Client address.
    pickUpPostalCode String <optional>

    Recolection address postal code.
    pickUpAddressReference String

    Client address references.
    pickUpPrimaryReference String

    Client address district.
    pickUpSecondaryReference String

    Client address province.
    pickUpNotes String

    Package recollection notes.
    pickUpContactName String <optional>

    Contact Names for the recollection.
    pickUpContactPhone String <optional>

    Contact phone number for the recollection.
    pickUpContactDocumentTypeID String <optional>

    Contact document type for the recollection. It can take the next values "DNI", "PAS", "PTP", "CR" and "CE".
    pickUpContactDocumentNumber String <optional>

    Contact ID for the recollection.
    pickUpContactEmail String <optional>

    Email for recollection notification.
    dropBranchID String <optional>

    Delivery warehouse code.
    dropAddress String

    Final client delivery address.
    dropPostalCode String <optional>

    Recollection address's postal code.
    dropAddressReference String

    Final client address's references.
    dropPrimaryReference String

    Final client address's district.
    dropSecondaryReference String

    Final client address's province.
    dropNotes String

    Package delivery notes.
    dropContactName String <optional>

    Person's contact names for delivery.
    dropContactPhone String <optional>

    Person's phone number for delivery
    dropContactDocumentTypeID String <optional>

    Person's document type for delivery. It can take the next values "DNI", "PAS", "PTP", "CR" and "CE".
    dropContactDocumentNumber String <optional>

    Person's ID document for delivery.
    dropContactEmail String <optional>

    Delivery notification Email.
    pickUpPoint Array.<Float> <optional>

    Coordinates array of picking address. Must have at least two elements: longitud and latitud.
    dropPoint Array.<Float> <optional>

    Coordinates array of delivery address. Must have at least two elements: longitud and latitud.
    packages Array.<Object> <optional>

    Order's packages list.

    Properties
    Name Type Attributes Description
    ClientPackageID String <optional>

    Package code.
    envelope String <optional>

    Package envelop type.
    weight Float <optional>

    Package weight.
    weightUnit String <optional>

    Weight unit.
    size Int <optional>

    Package size.
    quantity Int <optional>

    Number of packages.
    name String <optional>

    Package description.
    currency String <optional>

    Currency type.
    unitaryProductPrice Float <optional>

    Unitary package price.
    Throws:

    If the enterpriseKey or the tags correct name is not provided, the orders will not be created.

    Returns:
    Response -

    Response object with the orders creation status.

    fnTrackOrderChazkiPosition(code, enterpriseKey)

    This API allows us to see all the Chazki movements and the delivery state.

    Data must be sent using GET protocol.

    URL: https://us-central1-chazki-link-dev.cloudfunctions.net/fnTrackOrderChazkiPosition

    Parameters:
    Name Type Description
    code String

    Delivery track code.
    enterpriseKey String

    Client unique authorization code.
    Examples

    Data sent in cURL format

      curl --request GET 'https://us-central1-chazki-link-dev.cloudfunctions.net/fnTrackOrderChazkiPosition?code=XXXXXXXXX&enterpriseKey=YYYYYYYYYYYYYYY'
